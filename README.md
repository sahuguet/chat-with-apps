# Todo · Agent

A todo app with an LLM-powered chat interface.  
The frontend derives its API client and Claude tool definitions at runtime from the backend's `/openapi.json` — no code duplication between Python and JS.

## Architecture

```
FastAPI (main.py)
  └── SQLModel / Pydantic models   ← single source of truth for all shapes
        │
        └── /openapi.json          ← auto-generated by FastAPI
              │
              └── buildAPIFromSpec() in index.html
                    ├── todoAPI     ← fetch() functions, one per operation
                    └── TOOLS       ← Claude tool definitions via deriveTools()
```

## Setup

```bash
pip install fastapi uvicorn sqlmodel

mkdir static
cp index.html static/

uvicorn main:app --reload
```

Open http://localhost:8000, paste your Anthropic API key, and start chatting.

## How it works

On page load, `buildAPIFromSpec('/openapi.json')` walks every path+method in the
OpenAPI spec and constructs a `def()`-wrapped async `fetch()` function for each
operation:

- `operationId`       → function name (`list_todos`, `add_todo`, …)
- `summary`           → description sent to Claude
- `requestBody`       → input JSON Schema (used for validation + Claude tool definition)
- `responses[2xx]`    → output JSON Schema (used for local validation)
- `path` + `method`   → the actual `fetch()` call

`$ref`s are resolved inline so Claude always receives flat JSON Schema objects.

`deriveTools(todoAPI)` then maps this into the Anthropic tools array.  
The `input_schema` is the OpenAPI/JSON Schema object — no conversion needed.

## Adding a new endpoint

1. Add a route in `main.py` with an explicit `operation_id` and `summary`.
2. Restart the server.
3. Reload the page — the new tool appears automatically.

No JS changes required.
