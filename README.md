# chat-with-apps

An LLM-powered chat interface that works with any OpenAPI-described backend.
The frontend derives its API client and Claude tool definitions at runtime from
the backend's `/openapi.json` — no code duplication between backend and JS.

Includes a todo app as a reference implementation.

## Architecture

```
FastAPI (main.py)
  └── SQLModel / Pydantic models   ← single source of truth for all shapes
        │
        └── /openapi.json          ← auto-generated by FastAPI
              │
              └── buildAPIFromSpec() in chat-with-app.js
                    ├── API client  ← fetch() functions, one per operation
                    └── TOOLS       ← Claude tool definitions via deriveTools()
```

## Setup

```bash
pip install fastapi uvicorn sqlmodel httpx

export ANTHROPIC_API_KEY=sk-ant-...

uvicorn main:app --reload
```

Open http://localhost:8000, enter your Anthropic API key, and start chatting.

## The `<chat-with-app>` web component

The chat interface is packaged as a self-contained custom element that any page
can embed with two lines:

```html
<chat-with-app spec-url="/openapi.json"></chat-with-app>
<script type="module" src="/chat-with-app.js"></script>
```

### Attributes

| Attribute | Default | Description |
|---|---|---|
| `spec-url` | `/openapi.json` | URL of the OpenAPI spec to load |
| `system-prompt` | built-in | Override the agent's system prompt |

### Properties

| Property | Description |
|---|---|
| `apiKey` | Set the Anthropic API key programmatically (hides the key input) |
| `setBearerToken(token)` | Inject a Bearer token for authenticated API calls |

### Events

| Event | Detail | Description |
|---|---|---|
| `tool-executed` | `{ name, input, result }` | Fired after every tool call |

The host page listens for `tool-executed` to refresh its own UI after the agent
modifies data:

```js
document.getElementById('chatAgent').addEventListener('tool-executed', () => loadTodos());
```

### Style isolation

The component uses Shadow DOM — its styles do not affect the host page and
the host page's styles do not affect the component. CSS custom properties can
be set on the element to theme it.

## How it works

On connect, `buildAPIFromSpec(spec-url)` walks every path+method in the OpenAPI
spec and constructs an async `fetch()` function for each operation:

- `operationId`    → function name (`list_todos`, `add_todo`, …)
- `summary`        → description sent to Claude
- `requestBody`    → input JSON Schema (validation + Claude tool definition)
- `responses[2xx]` → output JSON Schema
- `path` + `method`→ the actual `fetch()` call

`$ref`s are resolved inline so Claude always receives flat JSON Schema objects.
`deriveTools()` maps the result into the Anthropic tools array — `input_schema`
is the OpenAPI/JSON Schema object with no conversion needed.

## Adding a new endpoint

1. Add a route in `main.py` with an explicit `operation_id` and `summary`.
2. Restart the server.
3. Reload the page — the new tool appears in the agent automatically.

No JS changes required.

## API key options

| Approach | Who pays | Setup |
|---|---|---|
| Browser input (default) | User | None — user pastes key into the component |
| Server-side proxy | App maker | Set `ANTHROPIC_API_KEY` env var; route calls through `/api/chat` |
| Chrome extension (planned) | User | Enter key once in extension, granted per origin |

The FastAPI backend already includes a `/api/chat` proxy endpoint. To use it,
change the fetch target in `chat-with-app.js` from the Anthropic API URL to
`/api/chat` and remove the API key input.

## Bearer token support

For APIs that require authentication, the component can inject a Bearer token
into all tool call requests without the agent ever seeing credentials:

```js
document.querySelector('chat-with-app').setBearerToken('your-token');
```

On Swagger pages (`/docs`), the token can be captured automatically after the
user authenticates via the Authorize dialog — see `NOTES-chrome-extension.md`
for the capture strategies.

## Chrome extension (planned)

A companion extension would make the chat available on any OpenAPI app without
any developer action:

1. User installs extension, enters Anthropic API key once.
2. Extension detects `/openapi.json` on every page visited.
3. `<chat-with-app>` is injected automatically — no embedding required.
4. On Swagger pages, Bearer tokens are captured after manual auth and silently
   injected into agent tool calls.

See `NOTES-chrome-extension.md` for the full design.
